# Рекомендації щодо покращення сайту (з акцентом на мобільну версію)

Аналіз проєкту (HTML, CSS, JS) виявив переваги та місця для доопрацювання. Нижче — пріоритизовані рекомендації.

---

## Критичні (вплив на мобільну версію)

### 1. Хедер: фіксовані піксельні позиції

**Проблема:** У `css/index.css` елементи хедера позиціоновані абсолютно з фіксованими `left` у пікселях:

- `.header .col-8` — `left: 749px`
- `.header .languages-block` — `left: 336px`
- `.header .custom-logo` — `left: 109px`
- `button.header__btn` — `left: 151px; top: 3px`

На екранах &lt; 768px це призводить до виїзду блоків за межі екрана або накладання.

**Рекомендація:** У медіа-запиті `@media (max-width: 767px)` (або 768px):

- Прибрати `position: absolute` та `left`/`top` у цих елементах.
- Використовувати flexbox: контейнер хедера вже має `.d-flex`, достатньо задати `position: relative` для `.header .container` / `.row` і не позиціонувати дочірні елементи абсолютно на мобільних.
- Кнопку «Як приєднатись?» вирівняти через `margin-left: auto` або порядок елементів у flex, без `left: 151px`.

Це відновить коректне розташування логотипу, заголовка та кнопки на малих екранах.

---

### 2. Фіксована висота секцій

**Проблема:**

```css
section.direction.section { height: 1266px; }
section.why-join.section  { height: 286px; }
```

На вузьких екранах контент може обрізатися або залишати великі порожні зони.

**Рекомендація:**

- Замінити на `min-height` там, де потрібна мінімальна висота, або на `height: auto` для секцій з динамічним контентом.
- Для мобільних додати, наприклад:  
  `@media (max-width: 767px) { section.direction.section, section.why-join.section { height: auto; min-height: 200px; } }`

---

### 3. Адаптивні зображення

**Проблема:** Контентні зображення (командири, напрямки, слайди) підключаються через один `src` без `srcset`/`sizes` або `<picture>`. Мобільні пристрої завантажують повнорозмірні файли.

**Рекомендація:**

- Додати `srcset` та `sizes` для ключових зображень (наприклад, галерея контактів, hero, напрямки).
- Для критичних зображень використати `<picture>` з варіантами для 1x/2x або різних ширини (наприклад, 480w, 768w, 1024w).
- Є окремий файл `bg-universe-mob-x2.webp` — переконатися, що він використовується в блоці з медіа-запитом або в `<picture>` для мобільних.

Це зменшить трафік і прискорить завантаження на мобільних.

---

## Важливі (UX і стабільність)

### 4. Safe area (вирізи та домашня кнопка)

**Проблема:** Не використовується `env(safe-area-inset-*)`. На пристроях з вирізом або округленнями контент може підходити під notch або індикатор жесту.

**Рекомендація:**

- Додати до фіксованого хедера, наприклад:  
  `padding-top: max(14px, env(safe-area-inset-top));`
- Для футера або кнопок внизу екрана:  
  `padding-bottom: max(1rem, env(safe-area-inset-bottom));`
- Можна винести в `:root`:  
  `--safe-top: env(safe-area-inset-top, 0);` і використовувати у потрібних місцях.

---

### 5. Відео в hero

**Що вже добре:** `preload="metadata"`, `playsinline`, `muted`, `autoplay`, `loop` — коректно для мобільних.

**Рекомендації:**

- Додати атрибут `poster` з невеликим зображенням (наприклад, перший кадр або окремий JPEG), щоб до старту відтворення була видима картинка.
- Перевірити розмір `20260128-compressed.mp4`; якщо файл дуже великий, можна підготувати коротший/легший варіант для мобільних і підключати його через `<source>` у `<picture>` або умовно через JS (на основі ширини екрана).

---

### 6. Модальне вікно форми (контакт/«Як приєднатись?»)

**Рекомендація:** На малих екранах зробити модалку повноекранною або майже повноекранною (наприклад, висота 100vh з урахуванням safe area), щоб форма була зручнішою і клавіатура не закривала поля. У Bootstrap можна додати клас типу `modal-fullscreen-sm-down` або власний клас і стилі для `@media (max-width: 575px)`.

---

### 7. Єдині брейкпоїнти

**Проблема:** Використовуються різні значення: 767px, 768px, 767.98px, 575.98px, 900px, 600px. Це ускладнює підтримку.

**Рекомендація:** Звести до 2–3 основних (наприклад, 576px, 768px, 992px або 1200px) і в подальшому використовувати їх у всіх медіа-запитах. Можна винести в CSS-змінні:

```css
:root {
  --bp-sm: 576px;
  --bp-md: 768px;
  --bp-lg: 992px;
}
@media (max-width: var(--bp-md)) { ... }
```

---

## Додаткові покращення

### 8. Розміри шрифтів на мобільних

- У блоці `@media (max-width: 768px)` для `.section-heading .lead` задано `font-size: 11px` — для другорядного тексту краще не нижче 12px (краще 13px) для читабельності.
- Перевірити інші дрібні тексти (підписи, підвальні посилання) і дотримуватися мінімуму ~12px для неосновного тексту.

---

### 9. Галерея контактів

**Що вже добре:** `-webkit-overflow-scrolling: touch`, `scroll-snap-type: x mandatory`, `scroll-snap-align: start`, на мобільних `max-width: min(280px, 85vw)` для карток.

**Рекомендація:** Переконатися, що картки в треку мають однакову ширину або передбачувану ширину для snap (наприклад, `scroll-snap-align: center` і однакова ширина карток), щоб при свайпі не було «напівкартки» на екрані.

---

### 10. Відступ для фіксованого хедера

**Проблема:** `body { margin-top: -80px; }` — контент піднімається під хедер. Висота хедера в стилях ~52–79px.

**Рекомендація:** Надати `main` або першій секції відступ зверху, рівний висоті хедера (наприклад, `padding-top: 80px` або змінна `--header-height`), щоб перший блок не ховався під хедером на всіх пристроях. На мобільних висота хедера може відрізнятися — варто перевірити і при потребі задати окремий padding для малих екранів.

---

### 11. Кнопка «Як приєднатись?» на мобільних

Зараз кнопка має фіксовані `width: 175px`, `height: 32px`, `font-size: 13px`. Для touch-пристроїв рекомендована мінімальна область натискання ~44×44px.

**Рекомендація:** У медіа-запиті для мобільних збільшити `min-height` до 44px і при потребі padding, залишивши ширину гнучкою (`min-width` замість фіксованої ширини), щоб зберегти зручне натискання.

---

### 12. Продуктивність

- Підключити критичні стилі inline або в одному файлі для above-the-fold контенту; решту підвантажувати з `media="print" onload="this.media='all'"` або асинхронно.
- Скрипти, що не потрібні для першого рендеру (слайдери, аналітика), підключати з `defer` або `async`.
- Переконатися, що шрифти (UAF Sans, Mulish тощо) підключені з `font-display: swap` і за потреби preload для ключових face.

---

## Короткий чеклист

| Задача | Пріоритет |
|--------|-----------|
| Прибрати фіксовані `left`/`top` у хедері на мобільних, вирівняти через flex | Високий |
| Секції direction/why-join: `height: auto` або `min-height` на малих екранах | Високий |
| Додати srcset/sizes або picture для основних зображень | Високий |
| Safe area для хедера/футера | Середній |
| Poster для hero-відео | Середній |
| Повноекранна/адаптивна модалка форми на мобільних | Середній |
| Єдині брейкпоїнти (змінні) | Низький |
| Мінімум 12–13px для другорядного тексту, 44px для кнопок | Середній |
| Padding-top для main під фіксований хедер | Середній |

Якщо потрібно, можу запропонувати конкретні фрагменти CSS/HTML для будь-якого з пунктів (наприклад, тільки хедер або тільки секції).
