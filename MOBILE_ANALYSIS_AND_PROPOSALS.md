# Аналіз мобільної версії та пропозиції покращень

Документ базується на аналізі коду проєкту, існуючих рекомендаціях у `MOBILE_RECOMMENDATIONS.md` та відомих підходах (WCAG, Apple HIG, Google Mobile, практики з open-source проєктів на GitHub).

---

## 1. Поточний стан (що вже добре)

| Область | Статус |
|--------|--------|
| **Viewport** | `width=device-width, initial-scale=1.0, viewport-fit=cover` — коректно |
| **Safe area** | Є `--safe-top`, `--safe-bottom`, використовуються в хедері та секціях |
| **Брейкпоїнти** | Оголошені змінні `--bp-sm`, `--bp-md`, `--bp-lg` (576, 768, 992px) |

## 2. Пропозиції покращень (пріоритизовані)

### Високий пріоритет

#### 2.1. Мінімальний розмір touch-таргетів (44×44px)

**Стандарт:** Apple HIG та WCAG 2.2 рекомендують мінімум 44×44 CSS px для клікабельних елементів.

**Що перевірити:**
- Кнопка «Як приєднатись?» на мобільній: зараз `min-height: 34px` — варто підняти до **44px** (або залишити 34px, але збільшити зону натискання через `padding` або псевдоелемент).
- Іконки соцмереж над відео — переконатися, що клікабельна область не менша за 44px.
- Плюсики в картках напрямків: контейнер `.params-link` 44×44px — ок; переконатися, що сама кнопка не обрізає hit area.

**Пропозиція:** У медіа-запиті для мобільних задати для всіх інтерактивних елементів (кнопки, посилання-іконки) `min-height: 44px; min-width: 44px` або еквівалент через padding, зберегши візуальний розмір через `box-sizing`.

---

#### 2.2. Мінімальний розмір шрифту для читабельності

**Стандарт:** Не менше 16px для основного тексту (усуває автоматичний zoom на iOS у полях вводу); для другорядного тексту — не нижче 12px (краще 13–14px).

**Поточний стан:** Є значення 10px (кнопка хедера), 11px, 12px у різних блоках.

**Пропозиція:**
- Основний текст і параграфи: не нижче **14px** на мобільних.
- Другорядний текст (підписи, lead): мінімум **12px**, краще **13px**.
- Кнопка в хедері: якщо залишати 10px для довгого тексту — переконатися, що контраст достатній (WCAG AA). Інакше підняти до 11–12px і при потребі трохи скоротити напис (наприклад, «Приєднатись» замість «Як приєднатись?» тільки на дуже вузьких екранах).

---

#### 2.3. Єдине використання брейкпоїнтів через змінні

**Проблема:** У проєкті використовуються і 768px, і 576px, і 600px, і 1200px без завжди однакових значень. Це ускладнює підтримку та тестування.

**Пропозиція:** У всіх нових і поступово в існуючих медіа-запитах використовувати змінні:

```css
@media (max-width: var(--bp-md)) { ... }
@media (max-width: var(--bp-sm)) { ... }
@media (min-width: var(--bp-lg)) { ... }
```

Додати при потребі `--bp-xs: 375px` для дуже малих екранів і використовувати його для екстремально компактних правок.

---

### Середній пріоритет

#### 2.4. Адаптивні зображення (srcset для 2x та ширини)

**Поточний стан:** Галерея командирів має `srcset="... 1x"` та `sizes="(max-width: 768px) 85vw, 320px"` — це добре, але немає варіантів для 2x (Retina).

**Пропозиція:**
- Для hero/posters та ключових зображень додати `srcset` з дескрипторами `1x`, `2x` або `480w`, `768w` після підготовки відповідних файлів.
- Використати `bg-universe-mob-x2.webp` у блоці з фоном для мобільних через медіа-запит або `<picture>`.

---

#### 2.5. Скорочення контенту при дуже вузьком екрані (опційно)

**Ідея:** На екранах < 360px можна трохи скоротити текст кнопки хедера (наприклад, показувати «Приєднатись» замість «Як приєднатись?») через окремий клас або `data-*` атрибут і CSS/мінімальний JS, щоб усе стабільно вміщувалось в один ряд без переносу.

---

#### 2.6. Відступ під фіксований хедер

**Поточний стан:** Є `padding-top: max(80px, calc(80px + var(--safe-top)))` для першої секції. Висота хедера на мобільній — 56px.

**Пропозиція:** Винести висоту хедера в змінну, наприклад `--header-height: 56px`, і для мобільних використовувати її в `padding-top` першої секції, щоб не було розбіжності (80px vs 56px). На ПК залишити більше значення під відповідний хедер.

---

#### 2.7. Галерея: однакова ширина карток для snap

**Поточний стан:** Scroll-snap і `max-width: min(280px, 85vw)` для карток.

**Пропозиція:** Задати карткам фіксовану ширину в межах контейнера (наприклад, `width: 280px` на мобільних) або `min-width`/`max-width` однакові для всіх, щоб при свайпі snap завжди показував цілу картку без «напівкартки» по краях.

---

### Низький пріоритет / довгостроково

#### 2.8. Продуктивність

- Критичний CSS для above-the-fold — inline або один файл; решту підвантажувати асинхронно.
- Скрипти з `defer` / `async` там, де це можливо.
- Шрифти: `font-display: swap` та при потребі `preload` для ключових face.

#### 2.9. Доступність (a11y)

- Переконатися, що всі інтерактивні елементи мають доступний текст або `aria-label`.
- Контраст тексту та кнопок — перевірка за WCAG AA (наприклад, помаранчевий на темному фоні).
- Фокус при скроллі/табі — видимі `:focus-visible` стилі для кнопок і посилань.

#### 2.10. Тестування на пристроях

- Перевірка на реальних пристроях (iOS Safari, Android Chrome) з різними ширинами (320px, 375px, 414px).
- Перевірка з увімкненим збільшенням шрифту в системі (Accessibility).
- Перевірка з вирізом (notch) та gesture bar — safe area вже врахована, варто лише візуально перевірити.

## 4. Посилання на методи та стандарти

- **WCAG 2.2** (Target Size): мінімум 24×24 CSS px, рекомендовано 44×44 для touch.
- **Apple HIG** (Layout): мінімальні розміри для інтерактивних елементів, safe area.
- **Google Mobile-Friendly:** viewport, читабельний текст без zoom, достатній розмір таргетів.
- **MDN / web.dev:** responsive images (srcset, sizes, picture), responsive design.
- **Bootstrap 5 breakpoints:** 576, 768, 992, 1200px — узгоджено з вашими --bp-*.

Якщо потрібно, можу запропонувати конкретні патчі (фрагменти CSS/HTML) для будь-якого пункту з таблиці вище.

## 5. Впроваджено (після аналізу)

- **--header-height** у `:root` (56px) та для desktop (80px); перша секція використовує `var(--header-height)` для `padding-top`.
- **Кнопка «Як приєднатись?»** на мобільних: `min-height: 44px`, `min-width: 44px`, `font-size: 11px`, `padding: 8px 12px` (touch-таргет ≥44px).
- **Галерея командирів** на мобільних: картки з `width`/`max-width`/`flex: 0 0 min(280px, 85vw)` для стабільного scroll-snap.
- **Соц-іконки** над відео на екранах ≤576px: мінімум 44×44px для клікабельної зони.
